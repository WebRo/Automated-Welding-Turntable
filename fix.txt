# PART 1 — VERIFIED ISSUES (ARDUINO FIRMWARE)

## A. Blocking Code (CRITICAL)

### Issue 1: Homing is blocking

* Uses `while(...)` loop
* Stops entire system loop
* Prevents real-time handling of:

  * E-STOP
  * Serial commands
  * State updates

### Correct behavior:

* Homing must run inside State Machine (non-blocking)
* Loop must always continue executing

---

### Issue 2: Return-to-home is blocking

* Uses:
  `while (stepper.distanceToGo() != 0)`
* Freezes system during movement

### Correct behavior:

* Movement must be handled via state:

  * e.g. RETURNING_HOME state
  * checked in loop()

---

### Issue 3: delay() usage

* Found in:

  * small LED delay
* Even small delays are blocking

### Correct behavior:

* Replace with millis()-based timing
