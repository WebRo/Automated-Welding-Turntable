## 1. مقدمة عن المشروع والفائدة منه (Project Intro & Benefits)

**ما هو المشروع؟**
نظام أتمتة متكامل واحترافي للتحكم في طاولة لحام دوارة (Turntable) تعمل بمحرك خطوي (Stepper Motor). تم تصميم النظام ليعمل بشكل متزامن وذكي مع روبوت لحام تعاوني (Cobot)، بحيث يتخاطب الأردوينو مع الروبوت لضمان دقة اللحام وتوقيته.

**الفائدة من المشروع:**
- **زيادة الإنتاجية:** التشغيل الآلي المستمر حيث يمكن للطاولة العودة للصفر وتكرار الدورة دون تدخل بشري.
- **دقة متناهية:** تحكم دقيق بمواقع اللحام باستخدام واجهة تحكم مرئية بدلاً من البرمجة اليدوية المعقدة.
- **توفير الوقت:** يمكن للمشغل إعداد (تعليم) نقاط اللحام بسرعة وحفظها في ملفات لاستدعائها لاحقاً.
- **أمان عالٍ:** نظام مزود بإيقاف طوارئ، وحماية من التصادم، ومنع تداخل الأوامر البشرية مع العمل الآلي.

## 2. المكونات والمداخل والمخارج (Hardware & I/O)

### 2.1 الأطراف الرئيسية (Pin Map)

| المنفذ | الاتجاه | الجهاز              | الوظيفة                                           |
|:------:|:------:|---------------------|---------------------------------------------------|
| **5**  | خروج (OUT) | TB6600 — PUL+       | نبضات الخطوة للمحرك (السرعة والمسافة)               |
| **6**  | خروج (OUT) | TB6600 — DIR+       | تحديد اتجاه دوران المحرك                          |
| **7**  | خروج (OUT) | TB6600 — ENA+       | تفعيل/تعطيل المحرك (LOW=تشغيل، HIGH=فصل)         |
| **8**  | دخول (IN) | حساس الصفر (Sensor) | مستشعر نقطة الصفر (Homing Sensor)                |
| **9**  | خروج (OUT) | ريلاي → الروبوت     | نبضة 1 ثانية لإخبار الروبوت ببدء اللحام           |
| **10** | دخول (IN) | ريلاي ← الروبوت| استقبال إشارة من الروبوت (LOW = الروبوت جاهز/انتهى) |
| **11** | خروج (OUT) | NeoPixel WS2812     | مؤشر الحالة اللوني (LED ذكي)                   |

### 2.2 توصيل مشغّل المحرك TB6600 (Common Cathode)

```text
Arduino GND  ──→  PUL- و DIR- و ENA- (اربطهم معاً)
Arduino Pin 5  ──→  PUL+
Arduino Pin 6  ──→  DIR+
Arduino Pin 7  ──→  ENA+

12V/24V DC (+)  ──→  VCC
12V/24V DC (-)  ──→  GND
Motor Phase A → A+ و A-
Motor Phase B → B+ و B-
```
> ⚠️ **تحذير:** لا تفصل أسلاك المحرك (A/B) أثناء وصل الطاقة — قد يتلف الدرايفر فوراً!

### 2.3 ربط الروبوت (Cobot Interface)

- **إرسال للأمر (Pin 9):** نبضة لمدة ثانية واحدة تخبر الروبوت أن الطاولة وصلت للزاوية المطلوبة، ابدأ اللحام.
- **استقبال الأمر (Pin 10):** عندما يُنهي الروبوت عمله يرسل إشارة (LOW) للأردوينو لتنتقل الطاولة للنقطة التالية. (يُستخدم أيضاً لبدء الدورة الأولى).

### 2.4 مؤشر اللون (NeoPixel LED)

| اللون        | الحالة                                      |
|:------------:|---------------------------------------------|
| 🟢 أخضر      | جاهز (IDLE) أو تم التصفير بنجاح             |
| 🔵 أزرق      | المحرك يتحرك (Homing أو العودة للصفر أو تنفيذ برنامج) |
| 🟡 أصفر      | ARMED (ينتظر الروبوت) أو الروبوت يلحم أو المحرك متوقف يدوياً |
| 🔴 أحمر      | خطأ أو إيقاف طوارئ (يتطلب Homing جديد)    |

## 3. إعدادات المحرك والسرعة (Motor Settings & Speed)

يتميز النظام الآن بإمكانية **تغيير سرعة المحرك ديناميكياً** عبر واجهة Node-RED باستخدام شريط التمرير (Slider) من 10% إلى 100%. يتم حساب السرعات بناءً على القيم القصوى التالية:

```cpp
MAX_SPEED         = 1000.0  // نبضة/ثانية — أقصى سرعة مطلقة
MANUAL_SPEED      = 500.0   // سرعة الدوران اليدوي (CW/CCW) (قابلة للتعديل بالنسبة المئوية)
HOMING_SPEED      = 300.0   // سرعة البحث عن حساس الصفر (دائماً ثابتة وبطيئة للأمان)
PROGRAM_ACCELERATION = 3000.0 // تسارع عالٍ أثناء تنفيذ البرنامج لمنع التأخير
STEPS_PER_REVOLUTION = 200  // خطوات للدورة الكاملة
```

## 4. بروتوكول الاتصال السريال (Serial Protocol)

### 4.1 من Node-RED → Arduino (أوامر التحكم)

| الأمر | الوظيفة |
|-------|----------|
| `HOMING` | التحرك للبحث عن الحساس (مستشعر نقطة الصفر) |
| `SET_HOME_HERE` | تصفير العداد واعتبار الموقع الفعلي الحالي هو الصفر (بدون حساس) |
| `GO_HOME` | عودة المحرك بسرعة إلى نقطة الصفر الحالية المحددة |
| `MOVE_FORWARD` / `BACKWARD` | دوران يدوي مستمر (CW / CCW) |
| `HOLD` | إيقاف الدوران اليدوي وتفعيل إمكانية الحفظ |
| `ADD` | حفظ الموضع الحالي كقطة لحام |
| `SET_SPEED xxx` | ضبط سرعة المحرك (من 10 إلى 100) |
| `DISARM` | إلغاء وضع الاستعداد (الخروج من ARMED) |
| `ESTOP` | إيقاف طوارئ فوري وفصل المحرك |
| `START_PROGRAM 550 879 ...` | إرسال المصفوفة وبدء الاستعداد للعمل الآلي |

### 4.2 من Arduino → Node-RED (تحديثات الحالة)
يرسل الأردوينو رسائل مستمرة (كل 100ms) مثل `STEPS:1228` لتحديث الواجهة، ورسائل نصية توضيحية مثل `STATUS:MOVING_TO_HOME_POSITION` وغيرها لتوضيح حالة النظام للمشغل.

## 6. دورة العمل الكاملة (Full Workflow)

1. **التشغيل والتصفير:**
   - شغّل النظام وافتح الواجهة.
   - اضغط `HOMING` للبحث عن الحساس، أو حرك الطاولة للوضع المطلوب واضغط `SET HOME` لتحديد الصفر.
2. **البرمجة (التعليم):**
   - حرك الطاولة بزر `CW` للزاوية المراد لحامها.
   - اضغط `HOLD` للتوقف.
   - اضغط `ADD` لحفظ الزاوية. (كرر الخطوات لجميع زوايا القطعة).
3. **بدء العمل (التسليح):**
   - اضغط `START`. سيتم قفل الأزرار اليدوية وتُضيء لمبة الأردوينو بالأصفر.
   - الطاولة الآن `ARMED` وتنتظر إشارة الروبوت.
4. **التنفيذ الآلي:**
   - يرسل الروبوت إشارة (LOW) → الطاولة تتحرك لأول نقطة.
   - الطاولة تصل للنقطة → ترسل نبضة (1 ثانية) للروبوت ليقوم باللحام.
   - الروبوت ينهي اللحام ويرسل (LOW) → الطاولة تتحرك للنقطة التالية...
5. **العودة والتكرار:**
   - بعد الانتهاء من جميع النقاط، ترجع الطاولة للصفر تلقائياً.
   - تدخل في حالة `ARMED` مجدداً لتكون جاهزة للقطعة التالية بمجرد قيام الروبوت بإرسال إشارة البدء.

## 7. ملاحظات تقنية وهندسية (Technical Notes)

1. **Non-Blocking Architecture:** تمت إعادة كتابة الكود ليعمل بنظام (Non-Blocking). عمليات التصفير، والعودة للصفر، والدوران، جميعها لا توقف المعالج (No `while` loops for movement)، مما يعني أن الأردوينو يستمر في استقبال أوامر الطوارئ وتحديث الواجهة أثناء الحركة.
2. **إصلاح AccelStepper Target Bug:** الدوران اليدوي المستمر (runSpeed) تم فصله عن الحركة المطلقة (moveTo) باستخدام خدعة برمجية تعيد ضبط مسار المحرك لضمان التوافق التام ومنع تجمد المحرك عند طلب `GO_HOME`.
3. **حماية الواجهة (UI Guards):** الواجهة تمتلك نظام حماية دقيق (State-based UI) يقوم بتعطيل الأزرار التي قد تسبب أعطالاً ميكانيكية أو برمجية في الوقت غير المناسب (مثل منع تغيير السرعة أثناء اللحام، أو تعطيل CCW أثناء الحركة).
4. **Pin 10 (إشارة الروبوت):** مصمم بمقاومة رفع داخلية (INPUT_PULLUP) لمنع أي تشويش كهربائي ناتج عن بيئة اللحام المليئة بالضجيج الكهرومغناطيسي.
