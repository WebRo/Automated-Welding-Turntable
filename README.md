# توثيق مشروع طاولة اللحام الآلية
# Automated Welding Turntable — Full Documentation

**الإصدار:** 6.0  
**التاريخ:** أبريل 2026  
**اللوحة:** Arduino Mega 2560  
**الواجهة:** Node-RED Dashboard (Vue.js ui_template)

---

## 1. نظرة عامة على المشروع (Project Overview)

نظام أتمتة متكامل للتحكم في طاولة لحام دوارة (Turntable) تعمل بمحرك خطوي (Stepper Motor)، مُصمَّم للعمل بالتزامن مع روبوت لحام تعاوني (Cobot).

**فلسفة العمل:**
- المشغّل يُعدّ الأوضاع مسبقاً عبر واجهة Node-RED
- يضغط START مرة واحدة للتأكيد → النظام يُسلَّح وينتظر الروبوت
- الروبوت يتحكم في بدء كل دورة لحام تلقائياً
- بعد كل دورة، الطاولة تعود للصفر وتنتظر الروبوت مجدداً بدون تدخل المشغّل

## 2. المكونات والتوصيلات (Hardware & Wiring)

### 2.1 الأطراف الرئيسية (Pin Map)

| المنفذ | النوع  | الجهاز              | الوظيفة                                           |
|:------:|:------:|---------------------|---------------------------------------------------|
| **5**  | OUTPUT | TB6600 — PUL+       | نبضات الخطوة للمحرك                               |
| **6**  | OUTPUT | TB6600 — DIR+       | تحديد اتجاه دوران المحرك                          |
| **7**  | OUTPUT | TB6600 — ENA+       | تفعيل/تعطيل المحرك (LOW=تشغيل، HIGH=فصل)         |
| **8**  | INPUT_PULLUP | حساس الصفر   | مستشعر نقطة الصفر (Homing Sensor)                |
| **9**  | OUTPUT | ريلاي → الروبوت     | نبضة 1 ثانية لإخبار الروبوت ببدء اللحام           |
| **10** | INPUT_PULLUP | ريلاي ← الروبوت| استقبال إشارة من الروبوت (LOW = إشارة جاهزة)     |
| **11** | OUTPUT | NeoPixel WS2812     | مؤشر الحالة اللوني (LED واحدة)                   |

### 2.2 توصيل مشغّل المحرك TB6600 (Common Cathode)

```
Arduino GND  ──→  PUL-  (اربطهم معاً بكوبري)
Arduino GND  ──→  DIR-
Arduino GND  ──→  ENA-

Arduino Pin 5  ──→  PUL+   (نبضات الخطوة)
Arduino Pin 6  ──→  DIR+   (الاتجاه)
Arduino Pin 7  ──→  ENA+   (التفعيل)

12V/24V DC (+)  ──→  VCC   (مصدر الطاقة الخارجي)
12V/24V DC (-)  ──→  GND

Motor Phase A → A+ و A-
Motor Phase B → B+ و B-
```

> ⚠️ **تحذير:** لا تفصل أسلاك المحرك (A/B) أثناء وصل الطاقة — يتلف الدرايفر فوراً!

### 2.3 ربط الروبوت (Cobot Interface)

```
┌─────────────────────────────────────────────────────┐
│                    Arduino Mega                      │
│                                                      │
│  Pin 9  (OUTPUT) ──→ [Relay NO] ──→ Cobot Input     │
│                       نبضة 1 ثانية = ابدأ اللحام    │
│                                                      │
│  Pin 10 (INPUT)  ←── Cobot Output ──→ Arduino       │
│                       LOW = انتهى اللحام             │
│                       LOW = ابدأ دورة جديدة          │
└─────────────────────────────────────────────────────┘
```

**ملاحظة مهمة:** المنفذ 10 يُستخدم لغرضين:
1. **أثناء ARMED** → LOW تعني "ابدأ دورة جديدة" (الروبوت جاهز)
2. **أثناء WAITING_COBOT** → LOW تعني "انتهيت من اللحام"

### 2.4 مؤشر اللون (NeoPixel LED)

| اللون        | الحالة                                      |
|:------------:|---------------------------------------------|
| 🟢 أخضر      | جاهز (IDLE) أو تم التصفير بنجاح             |
| 🔵 أزرق      | المحرك يتحرك (Homing أو تنفيذ برنامج)      |
| 🟡 أصفر      | ARMED (ينتظر الروبوت) أو الروبوت يلحم      |
| 🔴 أحمر      | خطأ أو إيقاف طوارئ (يتطلب Homing جديد)    |

## 5. حالات النظام (State Machine)

```
IDLE ──[HOMING]──→ HOMING ──(مستشعر يُفعَّل)──→ IDLE (isHomed=true)
  │
  ├──[MOVE_FORWARD/BACKWARD]──→ MANUAL_MOVING
  │                                    │
  │                              [HOLD]↓
  │                          MANUAL_DECELERATING
  │                                    │
  │                              (توقف)↓
  │                           MANUAL_HOLD ──[ADD]──→ IDLE
  │
  └──[START_PROGRAM]──→ ARMED ──(Robot LOW)──→ DELAY_BEFORE_START (2s)
                           ↑                            │
                           │                      (2s انتهت)↓
                    (دورة تنتهي)               PROGRAM_MOVING
                    (عودة للصفر)                     │
                           │                  (وصل للهدف)↓
                           │                  RELAY_ACTIVE (1s)
                           │                        │
                           │                  WAITING_COBOT
                           │                        │
                           │                (Robot LOW)↓
                           │              DELAY_BEFORE_MOVE (3s)
                           │                        │
                           │                  (نقطة تالية)──→ PROGRAM_MOVING
                           │                        │
                           └──────(آخر نقطة انتهت)──┘

[ESTOP/ERROR] من أي مكان ──→ ERROR (isHomed=false, يطلب Homing)
[DISARM] من ARMED ──→ IDLE (يطلب START من جديد)
```

## 6. أوامر التواصل مع الأردوينو (Serial Protocol)

### 6.1 من Node-RED → Arduino

| الأمر | الوظيفة |
|-------|----------|
| `HOMING` | بدء عملية التصفير والبحث عن نقطة الصفر |
| `MOVE_FORWARD` | دوران يدوي مستمر باتجاه CW |
| `MOVE_BACKWARD` | دوران يدوي مستمر باتجاه CCW |
| `HOLD` | إيقاف الدوران اليدوي |
| `ADD` | حفظ الموضع الحالي في مصفوفة الأردوينو |
| `DISARM` | إلغاء التسليح (الخروج من ARMED إلى IDLE) |
| `ESTOP` | إيقاف طوارئ فوري |
| `START_PROGRAM 550 879 1228` | بدء البرنامج مع قائمة الخطوات |

### 6.2 من Arduino → Node-RED

| الرسالة | المعنى |
|---------|--------|
| `SYSTEM_READY` | الأردوينو أُعيد تشغيله |
| `STEPS:1228` | الموضع الحالي بالخطوات (كل 100ms) |
| `HOMED` | تم التصفير بنجاح |
| `STATUS:PROGRAM_ARMED` | النظام مُسلَّح وجاهز لإشارة الروبوت |
| `STATUS:ARMED_WAITING_FOR_ROBOT_START_SIGNAL` | ينتظر LOW على Pin 10 |
| `STATUS:ROBOT_START_SIGNAL_RECEIVED` | استُلمت إشارة الروبوت |
| `STATUS:PROGRAM_EXECUTION_STARTED` | الدورة بدأت فعلاً |
| `STATUS:STARTING_IN_2_SECONDS` | تأخير ثانيتين قبل الحركة |
| `STATUS:MOVING_TO_POSITION_1` | يتحرك للنقطة رقم X |
| `STATUS:RELAY_ON_TO_COBOT` | إرسال إشارة للروبوت (ريلاي فعّال) |
| `STATUS:WAITING_FOR_COBOT_RESPONSE` | ينتظر رد الروبوت |
| `STATUS:COBOT_FINISHED_WAITING_3_SECONDS` | الروبوت انتهى، انتظار 3 ثوانٍ |
| `STATUS:CYCLE_COMPLETE_PIECE_DONE` | دورة واحدة اكتملت |
| `STATUS:RETURNING_TO_HOME` | يعود للصفر |
| `STATUS:DISARMED_BY_OPERATOR` | تم إلغاء التسليح بتدخل المشغّل |
| `STATUS:SYSTEM_REQUIRES_NEW_HOMING` | يتطلب Homing جديداً |
| `ERROR:EMERGENCY_STOP_ACTIVATED` | توقف طوارئ |
| `ERROR:HOMING_TIMEOUT` | مستشعر الصفر لم يُستجب في 30 ثانية |
| `ERROR:WELD_TIMEOUT_POSITION_X` | الروبوت لم يرد في دقيقتين |

## 7. واجهة Node-RED — أزرار لوحة التحكم

### 7.1 القسم الأيمن (لوحة التحكم)

| الزر | اللون | الوظيفة | متى يكون نشطاً |
|------|:-----:|----------|----------------|
| **CCW** | برتقالي | دوران يدوي عكس عقارب الساعة | بعد Homing، خارج PROGRAM_RUNNING |
| **CW** | أخضر | دوران يدوي مع عقارب الساعة | بعد Homing، خارج PROGRAM_RUNNING |
| **HOLD** | بني | إيقاف الدوران اليدوي | عندما يكون المحرك يتحرك |
| **ZERO** | بنفسجي | بدء عملية التصفير (Homing) | عندما لا يتحرك المحرك، خارج PROGRAM_RUNNING |
| **ADD** | أزرق | حفظ الموضع الحالي في الجدول | بعد HOLD وبعد Homing |
| **START** | أخضر | تسليح النظام وإرسال الأوضاع للأردوينو | بعد Homing، بوجود أوضاع، وليس مُسلَّحاً |
| **E-STOP** | أحمر | إيقاف طوارئ فوري — يحذف كل الأوضاع | **دائماً** |

### 7.2 القسم الأيسر (جدول الأوضاع)

| الزر | الوظيفة | متى يكون نشطاً |
|------|----------|----------------|
| **↑ / ↓** | إعادة ترتيب الأوضاع | خارج PROGRAM_RUNNING |
| **EDIT / SAVE** | تعديل عدد الخطوات يدوياً | خارج PROGRAM_RUNNING |
| **DELETE** | حذف وضع | خارج PROGRAM_RUNNING |
| **SAVE** | حفظ الأوضاع كملف JSON | خارج PROGRAM_RUNNING |
| **LOAD** | تحميل أوضاع من ملف JSON | خارج PROGRAM_RUNNING |

### 7.3 سلوك الأزرار في كل حالة

| الحالة | CCW/CW | HOLD | ADD | EDIT/DELETE/ORDER | START |
|--------|:------:|:----:|:---:|:-----------------:|:-----:|
| **IDLE** (جاهز) | ✅ يحرك | ✅ يوقف | ✅ يحفظ | ✅ | ✅ |
| **ARMED** (ينتظر روبوت) | ✅ يُلغي التسليح فقط* | ✅ يُلغي التسليح | ✅ يُلغي التسليح + يحفظ | ✅ يُلغي التسليح | ❌ |
| **PROGRAM_RUNNING** (دورة تعمل) | ❌ | ❌ | ❌ | ❌ | ❌ |

> *عند الضغط على CW/CCW أثناء ARMED → **يُلغي التسليح فقط بدون تحريك الموتور** → يجب ضغط START من جديد

## 8. دورة العمل الكاملة (Full Workflow)

### 8.1 التشغيل الأول

```
1. تشغيل النظام
2. ضغط ZERO → الطاولة تبحث عن المستشعر → تصفير الموضع للصفر
3. استخدام CW/CCW للتحرك → HOLD للتوقف → ADD لحفظ الوضع
   (تكرار للأوضاع المطلوبة، أو LOAD لتحميل ملف محفوظ)
4. ضغط START → النظام يُسلَّح ← يرسل الأوضاع للأردوينو
5. الطاولة تنتظر إشارة من الروبوت (LED أصفر)
```

### 8.2 الدورة الآلية (تتكرر تلقائياً)

```
الروبوت يرسل LOW على Pin 10
  ↓
انتظار 2 ثانية (START_DELAY_MS)
  ↓
الطاولة تتحرك للوضع الأول
  ↓
وصلت → ريلاي Pin 9 يُفعَّل لثانية واحدة → الروبوت يبدأ اللحام
  ↓
الطاولة تنتظر إشارة الروبوت (LOW على Pin 10)
  ↓
الروبوت أرسل LOW → انتظار 3 ثوانٍ (DELAY_BEFORE_MOVE)
  ↓
الطاولة تتحرك للوضع التالي ... (تتكرر العملية)
  ↓
آخر وضع انتهى → الطاولة ترجع للصفر
  ↓
ARMED تلقائياً → تنتظر إشارة الروبوت مجدداً (LED أخضر)
```

### 8.3 تعديل الأوضاع بين الدورات

```
عند ARMED (بين الدورات):
المشغّل يُعدّل/يحذف/يضيف وضع
  ↓
النظام يُلغي التسليح تلقائياً (DISARM → Arduino)
  ↓
يجب ضغط START من جديد للتأكيد
```

### 8.4 إيقاف الطوارئ

```
E-STOP في أي وقت:
  ↓
الأردوينو: إيقاف فوري للمحرك + فصل ريلاي الروبوت
Node-RED: حذف جميع الأوضاع المحفوظة
  ↓
يتطلب: Homing جديد + إعادة برمجة الأوضاع + START
```

## 9. تدفق Node-RED (Flow)

```
[Serial In] ──→ [ui_template (Vue.js Dashboard)] ──→ [Serial Out]
                        ↕ msg.payload
                   يستقبل: STEPS:xxx / STATUS:xxx / ERROR:xxx
                   يرسل:   HOMING / MOVE_FORWARD / START_PROGRAM 550 879 ...
```

**الحالات في Node-RED:**

| المتغير | الوصف |
|---------|-------|
| `currentSteps` | الموضع الحالي بالخطوات |
| `positions[]` | مصفوفة الأوضاع في الجدول |
| `isHomed` | هل تم التصفير؟ |
| `isMoving` | هل المحرك يتحرك يدوياً؟ |
| `isHoming` | هل التصفير جارٍ؟ |
| `isProgramRunning` | هل الدورة تعمل فعلاً؟ |
| `isArmed` | هل النظام مُسلَّح وينتظر الروبوت؟ |

## 11. ملاحظات تقنية مهمة

1. **Homing = Blocking**: دالة `performHoming()` تستخدم `while loop` تُوقف الـ `loop()` الرئيسية — هذا مقصود لضمان دقة التصفير.

2. **Return to Home = Blocking**: عودة الطاولة للصفر بعد كل دورة أيضاً Blocking — لا مشكلة لأن البرنامج لن يستقبل أوامر خلالها.

3. **Pin 10 = مزدوج الاستخدام**: يُستخدم لإشارة "ابدأ" (في ARMED) ولإشارة "انتهيت" (في WAITING_COBOT).

4. **حفظ الأوضاع عند E-STOP**: الأردوينو لا يحذف `targetStepsArray[]` عند E-STOP، لكن Node-RED يحذف `positions[]` — هذا مقصود لإجبار المشغّل على مراجعة الأوضاع قبل START.

5. **أقصى عدد أوضاع**: 30 وضعاً (`targetStepsArray[30]`).
